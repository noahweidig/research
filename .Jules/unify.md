## 2026-03-11 - [End-of-Page Narrative Bridges]
**Learning:** When users reach the end of a long list of items (like publications), an abrupt end without a Call To Action leaves them without a clear next step. Also, ensuring that headers match navigation items unifies terminology.
**Action:** Always verify if a long scrolling page has a logical continuation or CTA at the bottom to maintain momentum, and explicitly align terminology across the page.

## 2026-03-11 - [Contextual Search Terminology]
**Learning:** Functional UI components (like search) should use generic terminology (e.g., "results") when the list encompasses multiple formats, even if the overall page heading is simplified. Calling diverse formats like presentations or media coverage by a specific term (e.g., "publications") in search feedback creates semantic inconsistency.
**Action:** Unify terminology by standardizing search labels, placeholders, and screen reader announcements to use generic, inclusive terms like "Search" and "results".

## 2026-03-11 - [Inclusive Page Headings for Diverse Content]
**Learning:** Using a specific format term (like "Publications") as a page heading or navigation label creates a semantic mismatch when the page also includes other diverse content types like "Presentations", "Peer Reviews", or "Media Coverage". Additionally, using an inclusive term like "Research" better unifies the page terminology with CTAs such as "discussing research".
**Action:** When a page aggregates multiple types of scholarly or professional outputs, use an inclusive, generic heading (e.g., "Research", "Work") rather than a format-specific one to maintain content structure and narrative coherence.
